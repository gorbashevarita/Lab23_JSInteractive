# Лабораторная работа №23. Интерактивные веб-приложения на JavaScript

### Описание игры:
##### Правила игры
Два игрока ("X" и "O") по очереди ставят свой символ на поле 3×3. Побеждает тот, кто первым выстроит три своих символа по горизонтали, вертикали или диагонали. Если поле заполнено, а победителя нет — ничья.
##### Логика победы/поражения
Игрок победил если выстроил 3 одинаковых символов в ряд
Игрок проиграл если выиграл противник
В остальных случаях: ничья
##### Особенности реализации
Игра реализована с помощью HTML, CSS и JavaScript. 
Управление идёт через клики по клеткам поля и кнопку «Начать заново».
### Игровая механика:
##### Как определяется победитель?
Проверяется, совпадают ли символы в одной из выигрышных комбинаций. Если да - игра останавливается и выводится сообщение о победе.
##### Как обрабатываются клики?
Проверяется, не занята ли клетка и активна ли игра.
Если условия выполнены, то клетка заполняется символом текущего игрока.
После этого запускается проверка результата.
##### Как работает перезапуск?
При нажатии на кнопку "Начать заново" сбрасываются переменные состояния игры, очищаются клетки и обновляется текст статуса.
### Стуруктура данных:
##### Объяснить массив board
Хранит состояние поля. Пустая клетка — "", занятая — "X" или "O". Индексы массива соответствуют data-index клеток в HTML.
##### Объяснить условия победы
Массив из 8 комбинаций индексов клеток, при заполнении которых игрок побеждает.
##### Как хранится состояние игры?
currentPlayer (символ текущего игрока — "X" или "O")
gameActive (флаг активности игры — true или false)
board (состояние поля)
### Алгоритмы:
##### Алгоритм проверки победы
П все комбинации из winConditions.
Для каждой комбинации проверяем, совпадают ли символы в клетках.
Если нашли выигрышную комбинацию — устанавливаем roundWon = true и прерываем цикл.
##### Алгоритм определения ничьи
После хода проверяем, нет ли пустых клеток.
Если поле заполнено и нет победителя — объявляем ничью и останавливаем игру.
##### Алгоритм смены игрока
После каждого хода проверяем результат (победа, ничья).
Если игра продолжается — меняем currentPlayer на противоположный символ.
Обновляем текст статуса с текущим игроком.
